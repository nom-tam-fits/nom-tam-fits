47  /**
48   * A header-data unit (HDU) containing a compressed binary table. A compressed table is still a binary table but with
49   * some additional constraints. The original table is divided into groups of rows (tiles) and each tile is compressed on
50   * its own. The compressed data is then stored in the 3 data columns of this binary table (compressed, gzipped and
51   * uncompressed) depending on the compression type used in the tile. Additional data columns may contain specific
52   * compression options for each tile (i.e. compressed table row) individually. Table keywords, which conflict with those
53   * in the original table are 'saved' under standard alternative names, so they may be restored with the original table
54   * as appropriate.
55   * <p>
56   * Compressing a table HDU is typically a two-step process:
57   * </p>
58   * <ol>
59   * <li>Create a <code>CompressedTableHDU</code>, e.g. with {@link #fromBinaryTableHDU(BinaryTableHDU, int, String...)},
60   * using the specified number of table rows per compressed block, and compression algorithm(s)</li>
61   * <li>Perform the compression via {@link #compress()}</li>
62   * </ol>
63   * <p>
64   * For example to compress a binary table:
65   * </p>
66   * 
67   * <pre>
68   *   BinaryTableHDU table = ...
69   *   
70   *   // 1. Create compressed HDU with the
71   *   CompressedTableHDU compressed = CompressedTableHDU.fromBinaryTableHDU(table, 4, Compression.ZCMPTYPE_RICE_1);
72   *   
73   *   // 2. Perform the compression.
74   *   compressed.compress();
75   * </pre>
76   * <p>
77   * which of course you can compact into a single line as:
78   * </p>
79   * 
80   * <pre>
81   * CompressedTableHDU compressed = CompressedTableHDU.fromBinaryTableHDU(table, 4, Compression.ZCMPTYPE_RICE_1).compress();
82   * </pre>
83   * <p>
84   * The two step process (as opposed to a single-step one) was probably chosen because it mimics that of
85   * {@link CompressedImageHDU}, where further configuration steps may be inserted in-between. After the compression, the
86   * compressed table HDU can be handled just like any other HDU, and written to a file or stream, for example.
87   * </p>
88   * <p>
89   * The reverse process is simply via the {@link #asBinaryTableHDU()} method. E.g.:
90   * </p>
91   * 
92   * <pre>
93   *    CompressedTableHDU compressed = ...
94   *    BinaryTableHDU table = compressed.asBinaryTableHDU();
95   * </pre>
96   *
97   * @see CompressedTableData
98   */
99  @SuppressWarnings("deprecation")

104     /**
105      * Enables or disables using reversed compressed/uncompressed heap indices for when decompressing variable-length
106      * arrays (VLAs), as was prescribed by the original FITS 4.0 standard and the Pence et al. 2013 convention.
107      * <p>
108      * The <a href="https://fits.gsfc.nasa.gov/standard40/fits_standard40aa-le.pdf">FITS 4.0 standard</a> defines the
109      * convention of compressing variable-length columns in binary tables. On page 50 it writes:
110      * </p>
111      * <blockquote> 2. Append the VLA descriptors from the uncompressed table (which may be either Q-type or P-type) to
112      * the temporary array of VLA descriptors for the compressed table. </blockquote>
113      * <p>
114      * And the original <a href="https://fits.gsfc.nasa.gov/registry/tiletablecompression/tiletable2.0.pdf">Tiled-table
115      * convention</a> by W. Pence et al. also writes:
116      * </p>
117      * <blockquote> [...] we concatenate the array of descriptors from the uncompressed table onto the end of the
118      * temporary array of descriptors (to the compressed VLAs in the compressed table) before the 2 combined arrays of
119      * descriptors are compressed and written into the heap in the compressed table. </blockquote> However, it appears
120      * that the commonly used tools <code>fpack</code> / <code>funpack</code> provided in CFITSIO do just the reverse of
121      * this presciption. They store the uncopressed pointer <i>before</i> the compressed pointers. Because these tools
122      * are widely used we want to support files created or consumed by these tools. As such we allow to deviate from the
123      * FITS standard, and swap the order of the stored VLA indices inside compressed tables.
124      * 
125      * @param  value <code>true</code> if we should use VLA indices in compressed tables that follow the format
126      *                   described in the original Pence et al. 2013 convention, and also the FITS 4.0 standard as of
127      *                   2024 Mar 1; otherwise <code>false</code>. These original prescriptions define the reverse of
128      *                   what is actually implemented by CFITSIO and its tools <code>fpack</code> and
129      *                   <code>funpack</code>. (Our default is to conform to CFITSIO, and the expected revision of the
130      *                   standard to match).
131      * 
132      * @see          #hasOldStandardVLAIndexing()
133      * @see          #asBinaryTableHDU()
134      * 
135      * @since        1.19.1
136      * 
137      * @author       Attila Kovacs
138      */
139     public static void useOldStandardVLAIndexing(boolean value) {
140         reversedVLAIndices = value;
141     }

163     /**
164      * Prepare a compressed binary table HDU for the specified binary table. When the tile row size is specified with
165      * -1, the value will be set ti the number of rows in the table. The table will be compressed in "rows" that are
166      * defined by the tile size. Next step would be to set the compression options into the HDU and then compress it.
167      *
168      * @param  binaryTableHDU              the binary table to compress
169      * @param  tileRows                    the number of rows that should be compressed per tile.
170      * @param  columnCompressionAlgorithms the compression algorithms to use for the columns (optional default
171      *                                         compression will be used if a column has no compression specified). You
172      *                                         should typically use one or more of the enum values defined in
173      *                                         {@link Compression}. The FITS standard currently allows only the lossless
174      *                                         GZIP_1, GZIP_2, RICE_1, or NOCOMPRESS (e.g.
175      *                                         {@link Compression#ZCMPTYPE_GZIP_1} or
176      *                                         {@link Compression#ZCMPTYPE_NOCOMPRESS}.)
177      *
178      * @return                             the prepared compressed binary table HDU.
179      *
180      * @throws IllegalArgumentException    if any of the listed compression algorithms are not approved for use with
181      *                                         tables.
182      * @throws FitsException               if the binary table could not be used to create a compressed binary table.
183      * 
184      * @see                                Compression
185      * @see                                Compression#ZCMPTYPE_GZIP_1
186      * @see                                Compression#ZCMPTYPE_GZIP_2
187      * @see                                Compression#ZCMPTYPE_RICE_1
188      * @see                                Compression#ZCMPTYPE_NOCOMPRESS
189      * @see                                #asBinaryTableHDU()
190      * @see                                #useOldStandardVLAIndexing(boolean)
191      */
192     public static CompressedTableHDU fromBinaryTableHDU(BinaryTableHDU binaryTableHDU, int tileRows,
193             String... columnCompressionAlgorithms) throws FitsException {
194 
195         Header header = new Header();
196 
197         CompressedTableData compressedData = new CompressedTableData();
198         compressedData.setColumnCompressionAlgorithms(columnCompressionAlgorithms);
199 
200         int rowsPerTile = tileRows > 0 ? tileRows : binaryTableHDU.getData().getNRows();
201         compressedData.setRowsPerTile(rowsPerTile);
202 
203         Cursor<String, HeaderCard> headerIterator = header.iterator();
204         Cursor<String, HeaderCard> imageIterator = binaryTableHDU.getHeader().iterator();
205         while (imageIterator.hasNext()) {
206             HeaderCard card = imageIterator.next();
207             CompressedCard.restore(card, headerIterator);
208         }
209 
210         CompressedTableHDU compressedHDU = new CompressedTableHDU(header, compressedData);
211         compressedData.prepareUncompressedData(binaryTableHDU.getData());
212         compressedData.fillHeader(header);
213 
214         return compressedHDU;
215     }

276     /**
277      * Returns the number of table rows that are compressed in each table tile. This may be useful for figuring out what
278      * tiles to decompress, e.g. via {@link #asBinaryTableHDU(int, int)}, when wanting to access select table rows only.
279      * This value is stored under the FITS keyword ZTILELEN in the compressed header. Thus, this method simply provides
280      * a user-friendly way to access it. Note that the last tile may contain fewer rows than the value indicated by this
281      * 
282      * @return               the number of table rows that are compressed into a tile.
283      * 
284      * @throws FitsException if the compressed header does not contain the required ZTILELEN keyword, or it is &lt;= 0.
285      * 
286      * @see                  #asBinaryTableHDU(int, int)
287      * 
288      * @since                1.19
289      */
290     public int getTileRows() throws FitsException {
291         int n = getHeader().getIntValue(Compression.ZTILELEN, -1);
292         if (n <= 0) {
293             throw new FitsException("imnvalid or missing ZTILELEN header keyword");
294         }
295         return n;
296     }

312     /**
313      * Restores a section of the original binary table HDU by decompressing a selected range of compressed table tiles.
314      * The returned section will start at row index <code>fromTile * getTileRows()</code> of the full table.
315      * 
316      * @param  fromTile                 Java index of first tile to decompress
317      * @param  toTile                   Java index of last tile to decompress
318      * 
319      * @return                          The uncompressed binary table HDU from the selected compressed tiles.
320      * 
321      * @throws IllegalArgumentException If the tile range is out of bounds
322      * @throws FitsException            If there was an issue with the decompression.
323      * 
324      * @see                             #getTileRows()
325      * @see                             #getTileCount()
326      * @see                             #asBinaryTableHDU()
327      * @see                             #fromBinaryTableHDU(BinaryTableHDU, int, String...)
328      * @see                             #useOldStandardVLAIndexing(boolean)
329      */
330     public BinaryTableHDU asBinaryTableHDU(int fromTile, int toTile) throws FitsException, IllegalArgumentException {
331         Header header = getTableHeader();
332         int tileSize = getTileRows();
333         getData().setRowsPerTile(tileSize);
334 
335         if (fromTile < 0 || toTile > getTileCount() || toTile <= fromTile) {
336             throw new IllegalArgumentException(
337                     "illegal tile range [" + fromTile + ", " + toTile + "] for " + getTileCount() + " tiles");
338         }
339 
340         // Set the correct number of rows
341         int rows = getHeader().getIntValue(Compression.ZNAXISn.n(2));
342         header.addValue(Standard.NAXIS2, Integer.min(rows, toTile * tileSize) - fromTile * tileSize);
343 
344         BinaryTable data = BinaryTableHDU.manufactureData(header);
345         BinaryTableHDU tableHDU = new BinaryTableHDU(header, data);
346         getData().asBinaryTable(data, getHeader(), header, fromTile);
347 
348         return tableHDU;
349     }

410     /**
411      * Performs the actual compression with the selected algorithm(s) and options. When creating a compressed table HDU,
412      * e.g using the {@link #fromBinaryTableHDU(BinaryTableHDU, int, String...)} method, the HDU is merely prepared but

414      * compression. The design would allow for setting options between creation and compressing, but in this case there
415      * is really nothing of the sort.
416      * 
417      * @return               itself
418      * 
419      * @throws FitsException if the compression could not be performed
420      * 
421      * @see                  #fromBinaryTableHDU(BinaryTableHDU, int, String...)
422      * @see                  #useOldStandardVLAIndexing(boolean)
423      */

425         getData().compress(getHeader());
426         return this;

429     /**
430      * Obtain a header representative of a decompressed TableHDU.
431      *
432      * @return                     Header with decompressed cards.
433      *
434      * @throws HeaderCardException if the card could not be copied
435      *
436      * @since                      1.18
437      */
438     public Header getTableHeader() throws HeaderCardException {
439         Header header = new Header();
440 
441         header.addValue(Standard.XTENSION, Standard.XTENSION_BINTABLE);
442         header.addValue(Standard.BITPIX, ElementType.BYTE.bitPix());
443         header.addValue(Standard.NAXIS, 2);
444 
445         Cursor<String, HeaderCard> tableIterator = header.iterator();
446         Cursor<String, HeaderCard> iterator = getHeader().iterator();
447 
448         while (iterator.hasNext()) {
449             CompressedCard.backup(iterator.next(), tableIterator);
450         }
451 
452         return header;
453     }
