80       * (optional keywords) The value of these indexed keywords (where n ranges from 1 to ZNAXIS ) shall contain a
81       * positive integer representing the number o f pixels along axis n of the compression tiles. Each tile of pixels is
82       * compressed separately and stored in a row of a variable-length vector column in the binary table. The size of
83       * each image dimension (given by ZNAXISn ) is not required to be an integer multiple of ZTILEn, and if it is not,

239      * This dithering algorithm is identical to the SUBTRACTIVE DITHER 1 algorithm described above, ex- cept that any
240      * pixels in the floating-point image that are equa l to 0.0 are represented by the reserved value -2147483647 in
241      * the quantized integer array. When the i mage is subsequently uncompressed and unscaled, these pixels are restored
242      * to their original va lue of 0.0. This dithering option is useful if the zero-valued pixels have special
243      * significance to the da ta analysis software, so that the value of these pixels must not be dithered.
244      */
245     public static final String ZQUANTIZ_SUBTRACTIVE_DITHER_2 = "SUBTRACTIVE_DITHER_2";

247     /**
248      * Gzip is the compression algorithm used in the free GN U software utility of the same name. It was created by
249      * Jean-loup Gailly and Mark Adler and is based on the DEFLATE algorithm, which is a combination of LZ77 and Huffman
250      * coding. DEFLATE was intended as a replacement for LZW and other patent-encumbered data compression algor ithms
251      * which, at the time, limited the usability of compress and other popular archivers. Furt her information about
252      * this compression technique is readily available on the Internet. The gzip alg orithm has no associated parameters
253      * that need to be specified with the ZNAMEn and ZVALn keywords.
254      */
255     public static final String ZCMPTYPE_GZIP_1 = "GZIP_1";
256 
257     /**
258      * If ZCMPTYPE = ’GZIP 2’ then the bytes in the array of image pixel values are shuffled in to decreasing order of
259      * significance before being compressed with the gzip algorithm. In other words, bytes are shuffled so that the most
260      * significant byte of every pixel occurs first, in order, followed by the next most significant byte, and so on for
261      * every byte. Since the most significan bytes of the pixel values often have very similar values, grouping them
262      * together in this way often achieves better net compression of the array. This is usually especially effective
263      * when compressing floating-point arrays.
264      */
265     public static final String ZCMPTYPE_GZIP_2 = "GZIP_2";
266 
267     /**
268      * If ZCMPTYPE = ’RICE 1’ then the Rice algorithm is used to compress and uncompress the image pixels. The Rice
269      * algorithm (Rice, R. F., Yeh, P.-S., and Miller, W. H. 1993, in Proc. of the 9th AIAA Computing in Aerospace
270      * Conf., AIAA-93-4541-CP, American Institute of Aeronautics and Astronautics) is simple and very fast, compressing
271      * or decompressing 10 7 pixels/sec on modern workstations. It requires only enough memory to hold a single block of
272      * 16 or 32 pixels at a time. It codes the pixels in small blocks and so is able to adapt very quickly to changes in
273      * the input image statistics (e.g., Rice has no problem handling cosmic rays, bright stars, saturated pixels,
274      * etc.).
275      */
276     public static final String ZCMPTYPE_RICE_1 = "RICE_1";
277 
278     /**
279      * If ZCMPTYPE = ’PLIO 1’ then the IRAF PLIO (Pixel List) algorithm is used to compress and uncompress the image
280      * pixels. The PLIO algorithm was developed to store integer-valued image masks in a compressed form. Typical uses
281      * of image masks are to segment images into regions, or to mark bad pixels. Such masks often have large regions of
282      * constant value hence are highly compressible. The compression algorithm used is based on run-length encoding,
283      * with the ability to dynamically follow level changes in the image, allowing a 16-bit encoding to be used
284      * regardless of the image depth. The worst case performance occurs when successive pixels have different values.
285      * Even in this case the encoding will only require one word (16 bits) per mask pixel, provided either the delta
286      * intensity change between pixels is usually less than 12 bits, or the mask represents a zero floored step function
287      * of constant height. The worst case cannot exceed npix*2 words provided the mask depth is 24 bits or less.
288      */
289     public static final String ZCMPTYPE_PLIO_1 = "PLIO_1";

291     /**
292      * Hcompress is an the image compression package written by Richard L. White for use at the Space Telescope Science
293      * Institute. Hcompress was used to compress the STScI Digitized Sky Survey and has also been used to compress the
294      * preview images in the Hubble Data Archive. Briefly, the method used is: <br>
295      * 1. a wavelet transform called the H-transform (a Haar transform generalized to two dimensions), followed by<br>
296      * 2. quantization that discards noise in the image while retaining the signal on all scales, followed by 10<br>
297      * 3. quadtree coding of the quantized coefficients.<br>
298      * The technique gives very good compression for astronomical images and is relatively fast. The calculations are
299      * carried out using integer arithmetic and a re entirely reversible. Consequently, the program can be used for
300      * either lossy or lossless compression , with no special approach needed for the lossless case (e.g. there is no
301      * need for a file of residuals .)
302      */
303     public static final String ZCMPTYPE_HCOMPRESS_1 = "HCOMPRESS_1";

361     /**
362      * <p>
363      * The null pixels in integer images are flagged by a reserved BLANK value and will be preserved if a lossless
364      * compression algorithm is used. If the image is compressed with a lossy algorithm, however (e.g., H-Compress with
365      * a scale factor greater than 1), then some other technique must be used to identify the null pixels in the image.
366      * </p>
367      * <p>
368      * The recommended method of recording the null pixels when a lossy compression algorithm is used is to create an
369      * integer data mask with the same dimensions as the image tile. Set the null pixels to 1 and all the other pixels
370      * to 0, then compress the mask array using a lossless algorithm such as PLIO or GZIP. Store the compressed byte
371      * stream in a variable-length array column called ’NULL PIXEL MASK’ in the row corresponding to that image tile.
372      * The ZMASKCMP keyword should be used to record the name of the algorithm used to compress the data mask (e.g.,
373      * RICE 1). The data mask array pixels will be assumed to have the shortest integer datatype that is supported by
374      * the compression algorithm (i.e., usually 8-bit bytes).
375      * </p>
376      * <p>
377      * When uncompressing the image tile, the software must check if the corresponding compressed data mask exists with
378      * a length greater than 0, and if so, then uncompress the mask and set the corresponding undefined pixels in the
379      * image array to the appropriate value (as given by the BLANK keyword).
380      * </p>
381      */
382     public static final String NULL_PIXEL_MASK_COLUMN = "NULL_PIXEL_MASK_COLUMN";

402     /**

406      * astrometric measurements derived from the compressed image. Of course, the smoothing should never be applied when
407      * the image has been losslessly compressed with a scale factor (defined above) of 0 or 1.
408      */
409     public static final String SMOOTH = "SMOOTH";

411     private final FitsKey key;

415     Compression(VALUE valueType, String comment) {
416         this(valueType, comment, null);
417     }

419     Compression(VALUE valueType, String comment, IFitsHeader uncompressedKey) {
420         key = new FitsKey(name(), IFitsHeader.SOURCE.INTEGRAL, HDU.BINTABLE, valueType, comment);
421         this.uncompressedKey = uncompressedKey;
422         FitsKey.registerStandard(this);
423     }

425     @Override

427         return key;
428     }
